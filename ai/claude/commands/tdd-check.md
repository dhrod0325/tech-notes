# TDD Check

현재 프로젝트의 TDD 상태를 점검합니다.

## 수행 작업

1. 테스트 파일 존재 여부 확인
2. 테스트 실행 및 결과 분석
3. 테스트 커버리지 측정
4. TDD 원칙 준수 여부 검토

## 실행

다음 항목을 순서대로 점검해주세요:

### 1. 테스트 구조 확인

- `src/test/java` 디렉토리 구조 분석
- 단위 테스트(`*Test.java`)와 통합 테스트(`*IntegrationTest.java`, `*IT.java`) 분류
- 테스트 클래스와 프로덕션 클래스 매핑 확인

### 2. 테스트 실행

```bash
./gradlew test --info
```

또는

```bash
./mvnw test
```

### 3. 커버리지 리포트 생성

Gradle:
```bash
./gradlew jacocoTestReport
```

Maven:
```bash
./mvnw jacoco:report
```

### 4. 점검 항목

- [ ] 모든 테스트 통과 여부
- [ ] 라인 커버리지 80% 이상
- [ ] 브랜치 커버리지 70% 이상
- [ ] Given-When-Then 패턴 사용
- [ ] 테스트 메서드 네이밍 규칙 준수
- [ ] 적절한 테스트 더블 사용

### 5. 결과 리포트

점검 결과를 다음 형식으로 보고해주세요:

```
## TDD 점검 결과

### 테스트 현황
- 총 테스트 수: X
- 성공: X
- 실패: X
- 스킵: X

### 커버리지
- 라인: X%
- 브랜치: X%

### 개선 필요 사항
- 항목 1
- 항목 2

### 권장 사항
- 항목 1
- 항목 2
```
